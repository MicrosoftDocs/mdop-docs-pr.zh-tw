# Microsoft 進階群組原則管理 4.0 逐步指南


本逐步指南說明使用群組原則管理主控台（GPMC）和 Microsoft 高級群組原則管理（AGPM）的群組原則管理的高級技術。 AGPM 會增加 GPMC 的功能，提供：

-   將管理群組原則物件（Gpo）委派給多個群組原則管理員的標準角色，以及委派對生產環境中 Gpo 的存取權的功能。

-   [封存] 可讓群組原則管理員在 Gpo 部署到生產環境之前，建立及修改 Gpo。

-   回滾到封存中任何舊版 GPO 的功能，以及限制儲存在封存中的版本數。

-   Gpo 的存回和取出功能，以確保群組原則系統管理員不會無意中覆寫對方的工作。

-   能夠搜尋具有特定屬性的 Gpo，以及篩選顯示的 Gpo 清單。

## AGPM 案例概述


在這種情況下，您將針對 AGPM 中的每個角色使用個別的使用者帳戶，示範如何在具有多個群群組原則管理員且具有不同許可權等級的環境中管理群組原則。 具體來說，您將會執行下列工作：

-   使用網域系統管理員群組成員的帳戶，安裝 AGPM 服務器並將 AGPM 系統管理員角色指派給帳戶或群組。

-   使用您將指派 AGPM 角色的帳戶，安裝 AGPM 用戶端。

-   使用具有 AGPM 系統管理員角色的帳戶，設定 AGPM 並委派對 Gpo 的存取權，方法是指派角色給其他帳戶。

-   從擁有 [編輯者] 角色的帳戶，要求建立新的 GPO，然後使用擁有核准者角色的帳戶來進行核准。 使用 [編輯] 帳戶，將 GPO 從封存中核取，編輯 GPO，將該 GPO 檢查到封存，然後要求部署。

-   使用具有核准者角色的帳戶，查看 GPO 並將它部署到您的生產環境。

-   使用具有編輯者角色的帳戶，建立一個 GPO 範本，並使用它做為建立新 GPO 的起點。

-   使用具有核准者角色的帳戶，刪除並還原 GPO。

### AGPM 服務器需求

AGPM Server 4.0 需要 Windows Server2008R2、Windows Server2008、Windows7 和 GPMC （來自遠端伺服器管理工具（RSAT）），或 Windows Vista （在 RSAT 中安裝 GPMC）。 支援32位和64位版本。

在安裝 AGPM Server 之前，您必須是 Domain 系統管理員群組的成員，除非另有說明，否則必須存在下列 Windows 功能：

-   GPMC

    -   Windows Server2008R2 或 Windows Server2008：如果 GPMC 不存在，則會透過 AGPM 自動安裝。

    -   Windows7：您必須從 RSAT 安裝 GPMC，才能安裝 AGPM。 如需詳細資訊，請參閱[Windows 7 的遠端伺服器管理工具](https://go.microsoft.com/fwlink/?LinkID=131280)（ https://go.microsoft.com/fwlink/?LinkID=131280) 。

    -   Windows Vista 與 SP1：安裝 AGPM 前，您必須從 RSAT 安裝 GPMC。 如需詳細資訊，請參閱適用[于 Windows Vista 的遠端伺服器管理工具（含 Service Pack 1](https://go.microsoft.com/fwlink/?LinkID=116179) （） https://go.microsoft.com/fwlink/?LinkID=116179) 。

-   .NET Framework 3.5 或更新版本

    -   Windows Server2008R2 或 Windows7：如果未提供 .NET Framework 3.5 或更新版本，則會透過 AGPM 自動安裝 .NET Framework 3.5。

    -   Windows Server2008 或 Windows Vista （含 SP1）：安裝 AGPM 前，您必須先安裝 .NET Framework 3.5 或更新版本。

AGPM 服務器需要下列 Windows 功能，如果不存在，就會自動安裝這些功能：

-   WCF 啟用;非 HTTP 啟用

-   Windows 處理序啟用服務

    -   處理模型

    -   .NET 環境

    -   配置 Api

### AGPM 用戶端需求

您的 AGPM 用戶端4.0 需要 Windows Server2008R2、Windows Server2008、Windows7 和 GPMC （來自 RSAT），或 Windows Vista （在 RSAT 中安裝了 SP1 和 GPMC）。 支援32位和64位版本。 您可以在執行 AGPM 服務器的電腦上安裝 AGPM 用戶端。

AGPM 用戶端需要下列 Windows 功能，除非另有說明不存在，否則會自動安裝：

-   GPMC

    -   Windows Server2008R2 或 Windows Server2008：如果 GPMC 不存在，則會透過 AGPM 自動安裝。

    -   Windows7：您必須從 RSAT 安裝 GPMC，才能安裝 AGPM。 如需詳細資訊，請參閱[Windows 7 的遠端伺服器管理工具](https://go.microsoft.com/fwlink/?LinkID=131280)（ https://go.microsoft.com/fwlink/?LinkID=131280) 。

    -   Windows Vista 與 SP1：安裝 AGPM 前，您必須從 RSAT 安裝 GPMC。 如需詳細資訊，請參閱適用[于 Windows Vista 的遠端伺服器管理工具（含 Service Pack 1](https://go.microsoft.com/fwlink/?LinkID=116179) （） https://go.microsoft.com/fwlink/?LinkID=116179) 。

-   .NET Framework 3.0 或更新版本

    -   Windows Server2008R2 或 Windows7：如果未提供 .NET Framework 3.0 或更新版本，則會透過 AGPM 自動安裝 .NET Framework 3.5。

    -   Windows Server2008 或 Windows Vista （含 SP1）：如果不存在 .NET Framework 3.0 或更新版本，則會透過 AGPM 自動安裝 .NET Framework 3.0。

### 案例需求

開始這個案例之前，請先建立四個使用者帳戶。 在案例中，您會將下列其中一個 AGPM 角色指派給每個帳戶： AGPM 系統管理員（完全控制）、審核者、編輯者和檢閱者。 這些帳戶必須能夠傳送及接收電子郵件訊息。 針對具有 AGPM 系統管理員、核准者及（選擇性）編輯者角色的帳戶，指派**連結 gpo**許可權。

**記事** 
預設會將 [**連結 gpo** ] 許可權指派給網域管理員和企業系統管理員的成員。 若要將**連結 gpo**許可權指派給其他使用者或群組（例如擁有 AGPM 管理員或核准者角色的帳戶），請按一下該網域的節點，然後按一下 [**委派**] 索引標籤，選取 [**連結 Gpo**]，按一下 [**新增**]，然後選取您要指派許可權的使用者或群組。

### <a href="" id="bkmk-config1"></a>步驟1：安裝 AGPM 服務器

在這個步驟中，您會在將執行 AGPM 服務的成員伺服器或網網域控制站上安裝 [AGPM 服務器]，然後設定封存。 所有的 AGPM 作業都是透過此 Windows 服務來管理，並以服務的認證執行。 您可以將 AGPM 服務器所管理的封存放在該伺服器或同一林中的另一台伺服器上。

**若要在將裝載 AGPM 服務的電腦上安裝 AGPM 服務器**

1.  以網域系統管理員群組成員的帳戶登入。

2.  啟動 Microsoft 桌面優化套件 CD，然後依照螢幕上的指示進行，以選取 [**高級群組原則管理-伺服器**]。

3.  在 [**歡迎**] 對話方塊中，按一下 **[下一步**]。

4.  在 [ **Microsoft 軟體授權條款**] 對話方塊中，接受條款，然後按一下 **[下一步]**。

5.  在 [**應用程式路徑**] 對話方塊中，選取要安裝 [AGPM 服務器] 的位置。 安裝了 AGPM 服務器的電腦將會主持 AGPM 服務並管理封存。 按 **\[下一步\]**。

6.  在 [封存**路徑**] 對話方塊中，針對 AGPM 服務器選取要封存的位置。 封存路徑可以指向您在 AGPM 服務器或其他位置的資料夾。 不過，您應該選取一個位置，讓您有足夠的空間來儲存此 AGPM 服務器所管理的所有 Gpo 及歷程記錄資料。 按 **\[下一步\]**。

7.  在 [ **Agpm 服務帳戶**] 對話方塊中，選取 agpm 服務將在其下執行的服務帳戶，然後按一下 **[下一步]**。

    這個帳戶必須是網域系統管理員群組的成員，或是針對最低許可權設定，由 AGPM 服務器管理的每個網域中的下列群組：

    -   群組原則建立者擁有者

    -   備份運算子

    此外，這個帳戶需要下列資料夾的 [完全控制] 許可權：

    -   您的 AGPM 封存資料夾，此許可權會在安裝在本機磁片磁碟機上的 AGPM 服務器期間自動授與。

    -   本機系統 temp 資料夾，通常是%windir%\\temp。

8.  在 [封存**擁有**者] 對話方塊中，選取您指派 [AGPM 管理員] （完全控制）角色的帳戶或群組。 AGPM 管理員可以為其他群組原則管理員指派 AGPM 角色和許可權，以便在稍後您可以將 AGPM 系統管理員的角色指派給其他的群組原則系統管理員。 在此案例中，選取要在 AGPM 系統管理員角色中提供的帳戶。 按 **\[下一步\]**。

9.  在 [**埠**設定] 對話方塊中，輸入 AGPM 服務應該聆聽的埠。 除非您手動設定埠例外狀況，或使用規則來設定埠例外，否則請不要清除 [**將埠例外新增到防火牆**] 核取方塊。 按 **\[下一步\]**。

10. 在 [**語言**] 對話方塊中，選取一或多個要為 AGPM 服務器安裝的顯示語言。

11. 按一下 [**安裝**]，然後按一下 **[完成] 結束**設定向導。

    **小心** 請勿透過作業系統中的 [**管理工具**] 和 [**服務**] 來變更 AGPM 服務的設定。 執行此動作可能會使 AGPM 服務無法啟動。 如需如何變更服務設定的相關資訊，請參閱高級群組原則管理的說明。

     

### <a href="" id="bkmk-config2"></a>步驟2：安裝 AGPM 用戶端

每個群組原則管理員（任何建立、編輯、部署、評論或刪除 Gpo 的人）都必須在其所用的電腦上安裝 AGPM 用戶端，才能管理 Gpo。 您用來執行許多 GPO 管理工作的 [變更控制] 節點，只會在您安裝 AGPM 用戶端時，才會出現在 [群組原則管理] 主控台中。 在這種情況下，您必須在至少一部電腦上安裝 [AGPM 用戶端]。 您不需要在不執行「組原則管理」的最終使用者電腦上安裝 AGPM 用戶端。

**在群組原則系統管理員的電腦上安裝 AGPM 用戶端**

1.  啟動 Microsoft 桌面優化套件 CD，然後依照螢幕上的指示選取 [**高級組原則管理-用戶端**]。

2.  在 [**歡迎**] 對話方塊中，按一下 **[下一步**]。

3.  在 [ **Microsoft 軟體授權條款**] 對話方塊中，接受條款，然後按一下 **[下一步]**。

4.  在 [**應用程式路徑**] 對話方塊中，選取要安裝 AGPM 用戶端的位置。 按 **\[下一步\]**。

5.  在 [ **Agpm 伺服器**] 對話方塊中，輸入您要連接的 agpm 伺服器和埠的 DNS 名稱或 IP 位址。 AGPM 服務的預設埠是4600。 請勿清除 [**透過防火牆允許 Microsoft 管理主控台**] 核取方塊，除非您手動設定埠例外狀況，或使用規則來設定埠例外狀況。 按 **\[下一步\]**。

6.  在 [**語言**] 對話方塊中，選取一或多個要為 AGPM 用戶端安裝的顯示語言。

7.  按一下 [**安裝**]，然後按一下 **[完成] 結束**設定向導。

### <a href="" id="bkmk-config3"></a>步驟3：設定 AGPM 服務器連線

AGPM 會儲存每個受控群組原則物件（GPO）的所有版本，也就是在中央封存中，AGPM 提供變更控制的每個 GPO。 這可讓群組原則管理員在離線時查看和變更 Gpo，而不會立即影響每個 GPO 的部署版本。

在這個步驟中，您要設定 AGPM 服務器連線，並確保所有群組原則管理員都連線到相同的 AGPM 服務器。 （如需有關如何設定多個 AGPM 服務器的詳細資訊，請參閱高級群組原則管理的說明。）

**針對所有群組原則管理員設定 AGPM 服務器連線**

1.  在已安裝 AGPM 用戶端的電腦上，以您選取做為封存擁有者的使用者帳戶登入。 此使用者擁有 AGPM 系統管理員的角色（完全控制）。

2.  按一下 [**開始**]，指向 [系統**管理工具**]，然後按一下 [**群組原則管理**] 來開啟 GPMC。

3.  編輯套用至所有群組原則系統管理員的 GPO。

4.  在 [**群組原則管理編輯器**] 視窗中，按兩下 [**使用者**設定]、[**原則**]、[系統**管理範本**]、[ **Windows 元件**] 和 [ **AGPM**]。

5.  在 [詳細資料] 窗格中，按兩下 [ **agpm]：指定預設的 Agpm 伺服器（所有網域）**。

6.  在 [**屬性**] 視窗中，選取 [**啟用**]，然後輸入裝載封存之伺服器的 DNS 名稱或 IP 位址與埠（例如， **server.contoso.com:4600**）。 根據預設，AGPM 服務會使用埠4600。

7.  按一下 **[確定]**，然後關閉 [**群組原則管理編輯器**] 視窗。 更新群組原則時，會針對每個群組原則管理員設定 AGPM 服務器連線。

### <a href="" id="bkmk-config4"></a>步驟4：設定電子郵件通知

在 AGPM 系統管理員（完全控制）中，您可以指定要在編輯者嘗試建立、部署或刪除 GPO 時，傳送包含要求之電子郵件訊息之核准者和 AGPM 系統管理員的電子郵件地址。 您也可以決定要從哪個別名傳送郵件。

**若要設定 AGPM 的電子郵件通知**

1.  在 [**群組原則管理編輯器**] 中，流覽至 [**變更控制**] 資料夾 

2.  在 [詳細資料] 窗格中，按一下 [**網域委派**] 索引標籤。

3.  在 [**寄件者電子郵件地址**] 欄位中，輸入要傳送通知的 AGPM 電子郵件別名。

4.  在 [收**件者電子郵件地址**] 欄位中，輸入您要指派核准者角色的使用者帳戶的電子郵件地址。

5.  在 [ **SMTP 伺服器**] 欄位中，輸入有效的 SMTP 郵件伺服器。

6.  在 [**使用者名稱**] 和 [**密碼**] 欄位中，輸入具有 SMTP 服務存取權的使用者認證。 按一下 **\[Apply\]** (套用)。

### <a href="" id="bkmk-config5"></a>步驟5：委派存取權

作為 AGPM 系統管理員（完全控制），您可以委派網域層級對 Gpo 的存取權，將角色指派給每個群組原則管理員的帳戶。

**記事** 您也可以在 GPO 層級（而非網域層級）委派存取權。 如需詳細資訊，請參閱高級群組原則管理的說明。

 

**重要** 您應該限制 [群組原則建立者擁有者] 群組中的成員資格，讓它不能用來避開對 Gpo 的 [AGPM] 管理。 （在 [**群組原則管理主控台**] 中，按一下要管理 gpo 的林和網域中的 [**群組原則物件**]，按一下 [**委派**]，然後設定設定以符合貴組織的需求。）

 

**若要委派整個網域中所有 Gpo 的存取權**

1.  在 [**網域委派**] 索引標籤上，按一下 [**新增**] 按鈕，選取 [群組原則管理員] 的使用者帳戶作為 [核准者]，然後按一下 **[確定]**。

2.  在 [**新增群組或使用者**] 對話方塊中，選取要指派該角色給帳戶的**核准者**角色，然後按一下 **[確定]**。 （此角色包含檢閱者角色）。

3.  按一下 [**新增**] 按鈕，選取 [群組原則管理員] 的使用者帳戶做為 [編輯者]，然後按一下 **[確定]**。

4.  在 [**新增群組或使用者**] 對話方塊中，選取要將該角色指派給帳戶的 [**編輯**者角色]，然後按一下 **[確定]**。 （此角色包含檢閱者角色）。

5.  按一下 [**新增**] 按鈕，選取 [群組原則管理員] 的使用者帳戶作為 [審查員]，然後按一下 **[確定]**。

6.  在 [**新增群組或使用者**] 對話方塊中，選取**檢閱者**角色，只將該角色指派給該帳戶。

您必須完成下列步驟，才能使用 AGPM 來建立、編輯、審閱及部署 Gpo。 此外，您還會建立範本、刪除 GPO，並還原刪除的 GPO。

### <a href="" id="bkmk-manage1"></a>步驟1：建立 GPO

在有多個群組原則管理員的環境中，具有 [編輯者] 角色的人員可以要求建立新的 Gpo。 不過，該要求必須由擁有核准者角色的人員核准。

在這個步驟中，您會使用具有 Editor 角色的帳戶，以要求建立新的 GPO。 您可以使用擁有核准者角色的帳戶，核准此要求以建立 GPO。

**要求透過 AGPM 建立並管理新的 GPO**

1.  在已安裝 AGPM 用戶端的電腦上，以 AGPM 中指派了 [編輯者角色] 的使用者帳戶登入。

2.  在 [**群組原則管理] 主控台**樹狀結構中，按一下要管理 gpo 的林和網域中的 [**變更控制**]。

3.  以滑鼠右鍵按一下 [**變更控制**] 節點，然後按一下 [**新增受控制的 GPO**]。

4.  在 [**新的受控 GPO** ] 對話方塊中：

    1.  若要接收申請的複本，請在 [**抄送**] 欄位中輸入您的電子郵件地址。

    2.  輸入**MyGPO**做為新 GPO 的名稱。

    3.  輸入新 GPO 的批註。

    4.  按一下 [**建立即時**]，讓新的 GPO 在核准後立即部署到生產環境。 按一下 **\[提交\]**。

5.  當 [ **AGPM 進度**] 視窗顯示整體進度完成時，請按一下 [**關閉**]。 新的 GPO 會顯示在 [**擱置**] 索引標籤上。

**核准擱置的要求以建立 GPO**

1.  在已安裝 [AGPM 用戶端] 的電腦上，使用在 AGPM 中擁有核准者角色的使用者帳戶登入。

2.  開啟該帳戶的電子郵件收件匣，請注意，您已從 AGPM 別名收到一封電子郵件，並要求您建立 GPO。

3.  在 [**群組原則管理] 主控台**樹狀結構中，按一下要管理 gpo 的林和網域中的 [**變更控制**]。

4.  在 [**內容**] 索引標籤上，按一下 [**未決**] 索引標籤以顯示擱置中的 gpo。

5.  以滑鼠右鍵按一下 [ **MyGPO**]，然後按一下 [**核准**]。

6.  按一下 **[是]** 以確認核准，然後將 GPO 移至 [**受控**] 索引標籤。

### <a href="" id="bkmk-manage2"></a>步驟2：編輯 GPO

您可以使用 Gpo 來設定電腦或使用者設定，並將它們部署到許多電腦或使用者。 在這個步驟中，您會使用擁有 Editor 角色的帳戶來從封存中取出 GPO、離線編輯 GPO、將編輯過的 GPO 核取到封存，並要求將 GPO 部署到生產環境。 在這種情況下，您可以設定 GPO 中的設定，以要求密碼至少為八個字元長。

**從封存檢查 GPO 以進行編輯**

1.  在已安裝 AGPM 用戶端的電腦上，使用在 AGPM 中擁有 [編輯者角色] 的使用者帳戶登入。

2.  在 [**群組原則管理] 主控台**樹狀結構中，按一下要管理 gpo 的林和網域中的 [**變更控制**]。

3.  在 [詳細資料] 窗格的 [**內容**] 索引標籤上，按一下 [**受控**] 索引標籤以顯示受控 gpo。

4.  以滑鼠右鍵按一下 [ **MyGPO**]，然後按一下 **[取出]。**

5.  輸入要在已取出之 GPO 之歷程記錄中顯示的批註，然後按一下 **[確定]**。

6.  當 [ **AGPM 進度**] 視窗顯示整體進度完成時，請按一下 [**關閉**]。 在 [**受控**] 索引標籤上，GPO 的狀態會標示為 [**已取出**]。

**若要離線編輯 GPO 並設定最小密碼長度**

1.  在 [**受控**] 索引標籤上，以滑鼠右鍵按一下 [ **MyGPO**]，然後按一下 [**編輯**] 以開啟 [**群組原則管理編輯器**] 視窗，並變更 GPO 的離線複本。 在此案例中，請設定最小密碼長度：

    1.  在 [**電腦**設定] 底下，按兩下 [**原則**]、[ **Windows 設定**]、[**安全性設定**]、[**帳戶原則**] 和 [**密碼原則**]。

    2.  在 [詳細資料] 窗格中，按兩下 [**最小密碼長度**]。

    3.  在 [屬性] 視窗中，選取 [**定義這個原則設定**] 核取方塊，將字元數設為**8**，然後按一下 **[確定]**。

2.  關閉 [**群組原則管理編輯器**] 視窗。

**將 GPO 檢查到封存**

1.  在 [**受控**] 索引標籤上，以滑鼠右鍵按一下**MyGPO** ，然後按一下 [**存回**]。

2.  輸入批註，然後按一下 **[確定]**。

3.  當 [ **AGPM 進度**] 視窗顯示整體進度完成時，請按一下 [**關閉**]。 在 [**受控**] 索引標籤上，GPO 的狀態會標示為 [**已核**取]。

**向生產環境要求 GPO 部署**

1.  在 [**受控**] 索引標籤上，以滑鼠右鍵按一下**MyGPO** ，然後按一下 [**部署**]。

2.  因為此帳戶不是核准者或 AGPM 管理員，所以您必須提交部署的要求。 若要接收申請的複本，請在 [**抄送**] 欄位中輸入您的電子郵件地址。 輸入要顯示在 GPO 歷程記錄中的批註，然後按一下 [**提交**]。

3.  當 [ **AGPM 進度**] 視窗顯示整體進度完成時，請按一下 [**關閉**]。 **MyGPO**會顯示在 [**擱置**] 索引標籤上的 [gpo] 清單中。

### <a href="" id="bkmk-manage3"></a>步驟3：審查及部署 GPO

在這個步驟中，您會擔當核准者、建立報告並分析設定，以及變更 GPO 中的設定，以判斷是否應該核准。 在您評估 GPO 之後，您可以將它部署到生產環境，並將 GPO 連結至網域或組織單位（OU）。 當針對該域或組織單位中的電腦重新整理群組原則時，GPO 就會生效。

**若要查看 GPO 中的設定**

1. 在已安裝 AGPM 用戶端的電腦上，以 AGPM 中指派擁有者角色的使用者帳戶登入。 任何具有檢閱者角色（包含在所有其他角色中）的群組原則管理員，都可以查看 GPO 中的設定。

2. 開啟該帳戶的電子郵件收件匣，請注意，您已收到來自 AGPM 別名的電子郵件，並要求您要部署 GPO。

3. 在 [**群組原則管理] 主控台**樹狀結構中，按一下要管理 gpo 的林和網域中的 [**變更控制**]。

4. 在 [詳細資料] 窗格的 [**內容**] 索引標籤上，按一下 [**擱置**] 索引標籤

5. 按兩下 [ **MyGPO** ] 以顯示其歷程記錄。

6. 查看最新版本的 MyGPO 中的設定：

   1.  在 [歷程**記錄**] 視窗中，以滑鼠右鍵按一下包含最近時間戳記的 GPO 版本，按一下 [**設定**]，然後按一下 [ **HTML 報告**]，以顯示 GPO 設定的摘要。

   2.  在網頁瀏覽器中，按一下 [**全部顯示**] 以顯示 GPO 中的所有設定。 關閉瀏覽器。

7. 將最新版本的 MyGPO 與簽入到封存的第一個版本進行比較：

   1. 在 [歷程**記錄**] 視窗中，按一下包含最近時間戳記的 GPO 版本。 按住 CTRL，然後按一下**電腦版本**不是 * * \ * * * 的最舊 GPO 版本。

   2. 按一下 [**差異**] 按鈕。 [**帳戶原則/密碼原則**] 區段會以綠色醒目提示，並在前面加上**\ [+ \]**。 這表示設定只會在後一個 GPO 版本中設定。

   3. 按一下 [**帳戶原則/密碼原則**]。 [**最小密碼長度**] 設定也會以綠色醒目提示，並在前面加上**\ [+ \]**，指出它僅在後一個 GPO 版本中進行設定。

   4. 關閉網頁瀏覽器。

**將 GPO 部署到生產環境**

1.  在 [**擱置**] 索引標籤上，以滑鼠右鍵按一下 [ **MyGPO** ]，然後按一下 [**核准**]

2.  輸入要包含在 GPO 歷程記錄中的批註。

3.  按一下 **\[是\]**。 當 [ **AGPM 進度**] 視窗顯示整體進度完成時，請按一下 [**關閉**]。 GPO 已部署至生產環境。

**將 GPO 連結至網域或組織單位**

1.  在 GPMC 中，以滑鼠右鍵按一下您要套用您所設定之 GPO 的網域或組織單位（OU），然後按一下 [**連結現有的 gpo**]。

2.  在 [**選取 GPO** ] 對話方塊中，按一下 [ **MyGPO**]，然後按一下 **[確定]**。

### <a href="" id="bkmk-manage4"></a>步驟4：使用範本建立 GPO

在這個步驟中，您會使用具有 Editor 角色的帳戶來建立及使用範本。 該範本是 GPO 的靜態版本，用作建立新 Gpo 的起點。 雖然您無法編輯範本，但您可以根據範本建立新的 GPO。 範本適用于快速建立包含許多相同原則設定的多個 Gpo。

**根據現有的 GPO 建立範本**

1.  在已安裝 AGPM 用戶端的電腦上，使用在 AGPM 中指派了 [編輯者角色] 的使用者帳戶登入。

2.  在 [**群組原則管理] 主控台**樹狀結構中，按一下要管理 gpo 的林和網域中的 [**變更控制**]。

3.  在 [詳細資料] 窗格的 [**內容**] 索引標籤上，按一下 [**受控**] 索引標籤

4.  以滑鼠右鍵按一下 [ **MyGPO**]，然後按一下 [**另存為範本**]，建立包含目前 MyGPO 中所有設定的範本。

5.  輸入**MyTemplate**做為範本的名稱和批註，然後按一下 **[確定]**。

6.  當 [ **AGPM 進度**] 視窗顯示整體進度完成時，請按一下 [**關閉**]。 新範本隨即出現在 [**範本**] 索引標籤上。

**要求透過 AGPM 建立並管理新的 GPO**

1.  按一下 [**受控**] 索引標籤。

2.  以滑鼠右鍵按一下 [**變更控制**] 節點，然後按一下 [**新增受控制的 GPO**]。

3.  在 [**新的受控 GPO** ] 對話方塊中：

    1.  若要接收申請的複本，請在 [**抄送**] 欄位中輸入您的電子郵件地址。

    2.  輸入**MyOtherGPO**做為新 GPO 的名稱。

    3.  輸入新 GPO 的批註。

    4.  按一下 [**建立即時**]，讓新的 GPO 在核准後立即部署到生產環境。

    5.  針對 [ **GPO 範本**]，選取 [ **MyTemplate**]。 按一下 **\[提交\]**。

4.  當 [ **AGPM 進度**] 視窗顯示整體進度完成時，請按一下 [**關閉**]。 新的 GPO 會顯示在 [**擱置**] 索引標籤上。

使用獲指派核准者角色的帳戶來核准擱置的要求，以建立 GPO，就像您在[步驟1：建立 gpo](#bkmk-manage1)一樣。 MyTemplate 合併了您在 MyGPO 中設定的所有設定。 因為 MyOtherGPO 是使用 MyTemplate 建立的，所以第一次包含 MyGPO 在 MyTemplate 建立時所包含的所有設定。 您可以產生差異報告來比較 MyOtherGPO 與 MyTemplate，以進行確認。

**從封存檢查 GPO 以進行編輯**

1.  在已安裝 AGPM 用戶端的電腦上，使用在 AGPM 中指派了 [編輯者角色] 的使用者帳戶登入。

2.  以滑鼠右鍵按一下 [ **MyOtherGPO**]，然後按一下 **[取出]。**

3.  輸入要在已取出之 GPO 之歷程記錄中顯示的批註，然後按一下 **[確定]**。

4.  當 [ **AGPM 進度**] 視窗顯示整體進度完成時，請按一下 [**關閉**]。 在 [**受控**] 索引標籤上，GPO 的狀態會標示為 [**已取出**]。

**若要離線編輯 GPO 並設定帳戶鎖定時間**

1.  在 [**受控**] 索引標籤上，以滑鼠右鍵按一下 [ **MyOtherGPO**]，然後按一下 [**編輯**] 以開啟 [**群組原則管理編輯器**] 視窗，並變更 GPO 的離線複本。 在此案例中，請設定最小密碼長度：

    1.  在 [**電腦**設定] 底下，按兩下 [**原則**]、[ **Windows 設定**]、[**安全性設定**]、[**帳戶原則**] 和 [**帳戶封鎖原則**]。

    2.  在 [詳細資料] 窗格中，按兩下 [**帳戶鎖定時間**]。

    3.  在 [屬性] 視窗中，核取 [**定義此原則設定**]，將 [持續時間] 設定為**30**分鐘，然後按一下 **[確定]**。

2.  關閉 [**群組原則管理編輯器**] 視窗。

將 MyOtherGPO 簽入封存並要求部署，就像您在[步驟2：編輯 GPO](#bkmk-manage2)中所做的一樣。 您可以使用 [差異] 報告，將 MyOtherGPO 與 MyGPO 或 MyTemplate 進行比較。 任何包含檢閱者角色的帳戶（AGPM 系統管理員 \ [完全控制 \]、審核者、編輯者或檢閱者）都可以產生報告。

**比較 GPO 與另一個 GPO 及範本**

1.  若要比較 MyGPO 和 MyOtherGPO：

    1.  在 [**受控**] 索引標籤上，按一下 [ **MyGPO**]。 按 CTRL 鍵，然後按一下 [ **MyOtherGPO**]。

    2.  以滑鼠右鍵按一下 [ **MyOtherGPO**]，指向 [**差異**]，然後按一下 [ **HTML 報表**]。

2.  若要比較 MyOtherGPO 和 MyTemplate：

    1.  在 [**受控**] 索引標籤上，按一下 [ **MyOtherGPO**]。

    2.  以滑鼠右鍵按一下 [ **MyOtherGPO**]，指向 [**差異**]，然後按一下 [**範本**]。

    3.  選取 [ **MyTemplate**及**HTML 報表**]，然後按一下 **[確定]**。

### <a href="" id="bkmk-manage5"></a>步驟5：刪除及還原 GPO

在此步驟中，您會擔任核准者來刪除 GPO。

**刪除 GPO**

1.  在已安裝 AGPM 用戶端的電腦上，以已指派核准者角色的使用者帳戶登入。

2.  在 [**群組原則管理] 主控台**樹狀結構中，按一下要管理 gpo 的林和網域中的 [**變更控制**]。

3.  在 [**內容**] 索引標籤上，按一下 [**受控**] 索引標籤以顯示受控制的 gpo。

4.  以滑鼠右鍵按一下 [ **MyGPO**]，然後按一下 [**刪除**]。 按一下 [**從封存及生產刪除 GPO** ]，即可刪除在生產環境中的 [封存] 和 [已部署的 gpo] 版本。

5.  輸入要顯示在 GPO 的審核追蹤中的批註，然後按一下 **[確定]**。

6.  當 [ **AGPM 進度**] 視窗顯示整體進度完成時，請按一下 [**關閉**]。 該 GPO 隨即會從 [**受控**] 索引標籤中移除，並顯示在 [**回收站**] 索引標籤上，可供您還原或銷毀。

有時候，您可能會在刪除仍需要的 GPO 之後進行探索。 在此步驟中，您會擔任核准者來還原已刪除的 GPO。

**還原已刪除的 GPO**

1.  在 [**內容**] 索引標籤上，按一下 [**回收站**] 索引標籤以顯示已刪除的 gpo。

2.  以滑鼠右鍵按一下 [ **MyGPO**]，然後按一下 [**還原**]。

3.  輸入要顯示在 GPO 歷程記錄中的批註，然後按一下 **[確定]**。

4.  當 [ **AGPM 進度**] 視窗顯示整體進度完成時，請按一下 [**關閉**]。 該 GPO 隨即會從 [**回收站**] 索引標籤中移除，並顯示在 [**受控**] 索引標籤上。

    **記事** 將 GPO 還原到封存後並不會自動將它重新部署到生產環境。 若要將 GPO 傳回生產環境，請像在步驟3中一樣部署 GPO [：審查及部署 gpo](#bkmk-manage3)。

     

在編輯和部署 GPO 之後，您可能會發現對 GPO 所做的最近變更會造成問題。 在這個步驟中，您會擔任核准者來回滾到舊版的 GPO。 您可以返回 GPO 歷程記錄中的任何版本。 您可以使用 [批註] 和 [標籤] 來識別已知良好的版本，以及何時進行特定的變更。

**若要回滾至較舊版本的 GPO**

1.  在 [**內容**] 索引標籤上，按一下 [**受控**] 索引標籤以顯示受控制的 gpo。

2.  按兩下 [ **MyGPO** ] 以顯示其歷程記錄。

3.  以滑鼠右鍵按一下要部署的版本，按一下 [**部署**]，然後按一下 **[是]**。

4.  當**進度**視窗顯示整體進度完成時，請按一下 [**關閉**]。 在 [歷程**記錄**] 視窗中，按一下 [**關閉**]。

    **記事** 若要確認重新部署的版本是否為所需版本，請檢查兩個版本的差異報告。 在 GPO 的 [歷程**記錄**] 視窗中，選取兩個版本，以滑鼠右鍵按一下這些版本，指向 [**差異**]，然後按一下 [ **HTML 報表**] 或 [ **XML 報表**]。
