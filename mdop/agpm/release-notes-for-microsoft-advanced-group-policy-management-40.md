## <span data-ttu-id="d9f39-123">AGPM 4.0 的已知問題</span><span class="sxs-lookup"><span data-stu-id="d9f39-123">Known issues with AGPM 4.0</span></span>


### <span data-ttu-id="d9f39-124">[從生產中匯入] 命令不會將設定匯入已取出的 GPO</span><span class="sxs-lookup"><span data-stu-id="d9f39-124">Import from Production command does not import settings into a GPO that is checked out</span></span>

<span data-ttu-id="d9f39-125">如果您在生產環境中編輯 GPO，您必須從生產中匯入 GPO，以更新離線封存中的 GPO。</span><span class="sxs-lookup"><span data-stu-id="d9f39-125">If you edit a GPO in the production environment, you must import the GPO from production to update the GPO in the offline archive.</span></span> <span data-ttu-id="d9f39-126">匯**入生產**命令的目的是讓您在完成編輯之前先執行最終的生產備份，以便在必要時回滾到生產備份。</span><span class="sxs-lookup"><span data-stu-id="d9f39-126">The **Import from Production** command is intended to let you perform a final production backup before you finish editing so that you can roll back to the production backup if it is necessary.</span></span>

<span data-ttu-id="d9f39-127">如果當您執行匯**入生產**命令時，gpo 已取出，則不會將生產變更併入 GPO 的取出版本。</span><span class="sxs-lookup"><span data-stu-id="d9f39-127">If the GPO is checked out when you run the **Import from Production** command, then the production changes are not incorporated into the checked out version of the GPO.</span></span> <span data-ttu-id="d9f39-128">不過，即使該版本不能進行編輯，也會將此 GPO 的匯入版本新增到 GPO 的歷程記錄中。</span><span class="sxs-lookup"><span data-stu-id="d9f39-128">However, the imported version of the GPO is added to the history of the GPO even though that version is not available to be edited.</span></span> <span data-ttu-id="d9f39-129">GPO 簽入時，該版本會取代封存中的匯入版本，但會在 GPO 的歷程記錄中提供這兩者。</span><span class="sxs-lookup"><span data-stu-id="d9f39-129">When the GPO is checked in, that version will supersede the imported version in the archive, but both are available in the history of the GPO.</span></span>

<span data-ttu-id="d9f39-130">因應措施 **：** 在從生產中匯入 GPO 前，請務必先將它取出。</span><span class="sxs-lookup"><span data-stu-id="d9f39-130">**Workaround:** Make sure that the GPO is checked in before you import it from production.</span></span> <span data-ttu-id="d9f39-131">如果未在您匯入 GPO 前將其檢入，您可以使用 [**復原取出**] 命令來放棄變更，並回滾到您從生產匯入的 gpo 版本。</span><span class="sxs-lookup"><span data-stu-id="d9f39-131">If the GPO was not checked in before you imported it, you can use the **Undo Check Out** command to discard your changes and roll back to the version of the GPO that you imported from production.</span></span>

### <span data-ttu-id="d9f39-132">在使用多個網站 Active Directory 拓朴的環境中，已取出的 Gpo 無法針對數分鐘進行編輯</span><span class="sxs-lookup"><span data-stu-id="d9f39-132">Checked out GPOs cannot be edited for several minutes in an environment that uses a multiple site Active Directory topology</span></span>

<span data-ttu-id="d9f39-133">AGPM 使用用戶端/伺服器模型。</span><span class="sxs-lookup"><span data-stu-id="d9f39-133">AGPM uses a client/server model.</span></span> <span data-ttu-id="d9f39-134">AGPM 服務器和 AGPM 用戶端都會針對群組原則作業決定自己最近的網網域控制站。</span><span class="sxs-lookup"><span data-stu-id="d9f39-134">The AGPM Server and the AGPM Client each determine their own closest domain controller for Group Policy operations.</span></span> <span data-ttu-id="d9f39-135">當您使用 AGPM 用戶端取出 GPO 時，該伺服器實際上是從離線封存檢查 GPO 的 AGPM 服務器，以在 SYSVOL 資料夾中的暫存檔案夾中。</span><span class="sxs-lookup"><span data-stu-id="d9f39-135">When you check out a GPO by using an AGPM Client, it is actually the AGPM Server that checks the GPO out from the offline archive to a temporary folder in the SYSVOL folder.</span></span>

<span data-ttu-id="d9f39-136">如果 AGPM 服務器與 AGPM 用戶端位於不同的網站，則暫時取出的 GPO 可能不會出現在本機網站的網網域控制站上，長達幾分鐘，或長達30分鐘，因為 SYSVOL 複製延隔時間。</span><span class="sxs-lookup"><span data-stu-id="d9f39-136">If the AGPM Server and the AGPM Client are in different sites, then the temporary checked out GPO may not be present on the local site's domain controller for several minutes or up to 30 minutes due to SYSVOL replication latency.</span></span> <span data-ttu-id="d9f39-137">在這種情況下，您無法使用在 AGPM 用戶端上的 GPMC 來編輯已取出的 GPO，直到已取出的 GPO 發生 SYSVOL 複製為止。</span><span class="sxs-lookup"><span data-stu-id="d9f39-137">In this situation, you cannot edit the checked out GPO using the GPMC on an AGPM Client until SYSVOL replication of the checked out GPO has occurred.</span></span>

<span data-ttu-id="d9f39-138">因應措施 **：** 最佳做法是，您應該將 AGPM 用戶端放在與與其連接的 AGPM 服務器相同的網站中，這樣您就不需要等到進行 SYSVOL 複製，就可以編輯已取出的 GPO。</span><span class="sxs-lookup"><span data-stu-id="d9f39-138">**Workaround:** As a best practice, you should position AGPM Clients in the same site as the AGPM Server to which they connect so that you do not have to wait for SYSVOL replication to occur before you can edit a checked out GPO.</span></span>

### <span data-ttu-id="d9f39-139">如果您的帳戶沒有封存的許可權，則 AGPM 無法讀取備份限制</span><span class="sxs-lookup"><span data-stu-id="d9f39-139">AGPM cannot read the backup limit if your account does not have permissions for the archive</span></span>

<span data-ttu-id="d9f39-140">在 AGPM 用戶端上，如果您使用尚未委派給 AGPM 封存之許可權的帳戶登入，請啟動群組原則管理主控台（GPMC），然後按一下 [**變更控制權**]，您會收到下列錯誤訊息。</span><span class="sxs-lookup"><span data-stu-id="d9f39-140">On an AGPM Client, if you log on by using an account that has not been delegated permissions to the AGPM archive, start the Group Policy Management Console (GPMC), and then click **Change Control**, you receive the following error.</span></span>

``` syntax
Failed to read backup purge limit for this domain. 

The following error occurred: 
You do not have sufficient permissions to perform this operation. 
Microsoft.Agpm.AccessDeniedException (80070005)
```

<span data-ttu-id="d9f39-141">因應措施 **：** 請與 AGPM 系統管理員（完全控制）聯繫，並要求他們針對您的帳戶委派對 AGPM 的存取權。</span><span class="sxs-lookup"><span data-stu-id="d9f39-141">**Workaround:** Contact an AGPM Administrator (Full Control) and request that they delegate access to AGPM for your account.</span></span> <span data-ttu-id="d9f39-142">如果您是 AGPM 系統管理員，請使用指派了 AGPM 系統管理員角色的帳戶登入，這樣您就可以委派其他帳戶的存取權。</span><span class="sxs-lookup"><span data-stu-id="d9f39-142">If you are an AGPM Administrator, log on by using an account to which the AGPM Administrator role is assigned so that you can delegate access for the additional account.</span></span> <span data-ttu-id="d9f39-143">如需詳細資訊，請參閱在 AGPM 說明中的「委派網域層級的檔案存取」。</span><span class="sxs-lookup"><span data-stu-id="d9f39-143">For more information, see "Delegate Domain-Level Access to the Archive" in the AGPM Help.</span></span>
